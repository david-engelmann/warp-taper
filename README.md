# warp-taper

Evidence-recording toolkit for [warpdotdev/warp](https://github.com/warpdotdev/warp) PRs. Runs a scenario against a local Warp build, captures screen + logs, runs programmatic assertions, and emits a bundle ready to attach to a PR comment.

## Pipeline

```
build      cargo build -p warp against $WARP_SOURCE
deploy     launch warp-oss
record     screen capture + log tail during the scenario
evaluate   run the scenario's assertions
bundle     emit a PR-ready README.md referencing the captured artifacts
```

## Install

```sh
cargo install --path crates/warp-taper-cli
```

## Usage

```text
$ warp-taper --help
Record Warp behavior into PR-ready evidence bundles.

Usage: warp-taper <COMMAND>

Commands:
  run            Run the full pipeline against a scenario directory
  run-builtin    Run a built-in (Rust-authored) scenario by name. Use `list-builtins` to see what's available
  list-builtins  List built-in scenarios available to `run-builtin`
  describe       Print metadata for a built-in scenario without running it
  init           Print a starter Rust scenario file to stdout
  version        Print version
  help           Print this message or the help of the given subcommand(s)
```

### Discover built-in scenarios

```text
$ warp-taper list-builtins
mcp-log-rotation

$ warp-taper describe mcp-log-rotation
slug:      10874-mcp-log-rotation
title:     MCP log rotation kicks in at the size cap
ticket:    warpdotdev/warp#10874
expected:
  An MCP server's log file rotates after writing past the configured size
  threshold (10 MiB by default, 5 rotated copies = 60 MiB cap per server).
  The MCP server continues to operate normally during rotation: no error
  toasts, no dropped connections, no crashed processes. Before PR #10874
  the active log grew without bound.
mcp_log_paths:
  - ~/Library/Group Containers/2BBY89MBSN.dev.warp/Library/Application Support/dev.warp.Warp-Stable/mcp
assertions (3):
  - mcp_log_snapshot_captured
  - mcp_rotation_occurred
  - log_lacks
```

### Run a built-in scenario

```sh
warp-taper run-builtin mcp-log-rotation \
    --warp-source ~/personal/warp \
    --duration-ms 60000
```

Or a YAML-defined scenario directory (`metadata.yaml` + optional `assertions.sh`):

```sh
warp-taper run path/to/scenario-dir --warp-source ~/personal/warp
```

Output lands in `tapes/<slug>/` (gitignored). Override with `--tape-dir`.

### Scaffold a new scenario

```text
$ warp-taper init 12345-fancy-fix --title "Fancy fix" --ticket "warpdotdev/warp#12345"
//! Built-in scenario: Fancy fix.

use crate::assertion::{Assertion, McpLogSnapshotCaptured};
use crate::error::Result;
use crate::scenario::Scenario;
use crate::scenarios::Builtin;

pub fn _12345_fancy_fix() -> Result<Builtin> {
    let scenario = Scenario::builder("12345-fancy-fix")
        .title("Fancy fix")
        .ticket("warpdotdev/warp#12345")
        .expected("TODO: describe the expected behavior")
        .build()?;

    let assertions: Vec<Box<dyn Assertion>> = vec![
        Box::new(McpLogSnapshotCaptured),
    ];

    Ok((scenario, assertions))
}

#[cfg(test)]
mod tests {
    use super::*;

    #[test]
    fn builds() {
        _12345_fancy_fix().unwrap();
    }
}
```

Pipe into `crates/warp-taper-core/src/scenarios/` and register it in
[`scenarios/mod.rs`](crates/warp-taper-core/src/scenarios/mod.rs).

## Example output: a real recorded tape

A worked example is checked in at [docs/sample-tape/](docs/sample-tape/). It was generated by [`scripts/generate-sample-tape.sh`](scripts/generate-sample-tape.sh), which drives `warp-taper run` against the synthetic [`scripts/demo-rotation/`](scripts/demo-rotation/) binary — a small Rust program that simulates the **LLM-driven log rotation** flow proposed for `simple_logger` (warpdotdev/warp PR #10882, branch `david/advanced-log-rotation-poc`). The demo:

1. Writes log entries to an "active" MCP log file.
2. When the file crosses the configured size threshold (800 bytes here for demo purposes; 10 MiB in real warp), rotates `active.log` → `active.log.1`.
3. Emits a `.rotations.jsonl` event (Layer A — always on).
4. Calls LM Studio at `http://localhost:1234/v1/chat/completions` to summarize the rotated content, falling back to a canned line when LM Studio isn't reachable.
5. Writes the summary to `.summaries.jsonl` (Layer B — opt-in).

What the tape contains:

| File | Bytes | What it shows |
|---|---|---|
| [`docs/sample-tape/master.mov`](docs/sample-tape/master.mov) | ~1.2 MB | 4-second screen capture of the demo running |
| [`docs/sample-tape/README.md`](docs/sample-tape/README.md) | 2 KB | PR-ready bundle README |
| [`docs/sample-tape/logs/warp-oss.session.log`](docs/sample-tape/logs/warp-oss.session.log) | 1.5 KB | slice of the warp-oss log written during the recording |
| [`docs/sample-tape/logs/mcp/demo-mcp-server-abc123.log.1`](docs/sample-tape/logs/mcp/demo-mcp-server-abc123.log.1) | 808 B | the rotated-out log file |
| [`docs/sample-tape/logs/mcp/demo-mcp-server-abc123.log.rotations.jsonl`](docs/sample-tape/logs/mcp/demo-mcp-server-abc123.log.rotations.jsonl) | — | rotation event (Layer A) |
| [`docs/sample-tape/logs/mcp/demo-mcp-server-abc123.log.summaries.jsonl`](docs/sample-tape/logs/mcp/demo-mcp-server-abc123.log.summaries.jsonl) | — | LLM summary record (Layer B) |
| [`docs/sample-tape/stages/`](docs/sample-tape/stages/) | — | per-stage `01-build` / `02-deploy` / `03-record` / `04-evaluate` logs |

A rotation event from the run:

```json
{"active_path":"<sample-tape>/mcp/demo-mcp-server-abc123.log",
 "bytes_rotated":808,"event":"rotated",
 "rotated_path":"<sample-tape>/mcp/demo-mcp-server-abc123.log.1",
 "rotation_index":1,"timestamp":"2026-05-14T17:00:30Z"}
```

The matching summary record:

```json
{"input_bytes":808,"rotation_index":1,"source":"canned-fallback",
 "summary":"12 routine search/read tool calls; 2 rate-limit backoffs;
            1 upstream cache miss recovered via source fallback. No
            user-visible errors. Suggested follow-up: monitor cache hit rate."}
```

(With LM Studio running, `source` would be `"lm-studio"` and the summary would come from the local model instead of the canned fallback.)

### Regenerating the sample tape

```sh
# defaults: 4-second recording, 900×540 region at (100, 100)
scripts/generate-sample-tape.sh

# headless / no screen recording — produces everything except a real .mov
NO_VIDEO=1 scripts/generate-sample-tape.sh
```

Requires Screen Recording permission for the parent process (Terminal or VSCode) — macOS → System Settings → Privacy & Security → Screen & System Audio Recording.

## Authoring scenarios

Scenarios are pure Rust. Each one is a function that returns a `(Scenario, Vec<Box<dyn Assertion>>)`. See [crates/warp-taper-core/src/scenarios/](crates/warp-taper-core/src/scenarios/) for the reference implementation (`mcp_log_rotation.rs`).

YAML+bash scenarios are supported via the `ShellScriptAssertion` adapter; see [CONTRIBUTING.md](CONTRIBUTING.md) for the format.

## Requirements

macOS for real screen recording (`/usr/sbin/screencapture`). Use `--no-screencapture` on other platforms to fall back to the no-op recorder.
